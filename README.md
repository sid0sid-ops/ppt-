## 🚀 How to Run the Project

### Option 1 — Double Click (Easiest)
Double-click **`start-bioalign.bat`** in the project root.
This starts both the Tailwind CSS watcher and the HTTP server at `http://localhost:8080`.

### Option 2 — Terminal
```bash
# Open a terminal in the project folder, then:
cmd /c "npm run dev"
```
Open your browser at **http://localhost:8080**

> **Note:** If you get "port already in use", the server is already running — just open the browser.

## ✏️ How to Edit Slide Content

**The one file you need:** `assets_global/js/features/content.js`

This file is a JavaScript array called `slidesData`. Each item in the array is one slide.

### Slide Object Structure

```js
{
    id: 1,                        // Slide number (must be sequential)
    title: "Slide Title Here",    // Shows in sidebar and browser tab
    notes: "What you will say...",// Speaker notes in Presenter HUD
    content: [],                  // Legacy field, leave empty []
    figure: null,                 // Legacy field, leave null
    htmlContent: `...`            // ⭐ The actual slide HTML goes here
}
```

### The `htmlContent` Field

This is a JavaScript template literal (backtick string) containing raw HTML.
You have access to these helper functions inside `htmlContent`:

| Helper | Purpose | Example |
|--------|---------|---------|
| `PANEL(content)` | Wraps content in the standard glass slide panel | `${PANEL(html)}` |
| `HDR(icon, section, title)` | Creates the header row with icon and title | `${HDR(ICON_BOLT, 'Section 1 · Slide 2', 'My Title')}` |
| `BULLETS(items)` | Creates a styled bullet list | See example below |
| `IMG(num, label)` | Embeds a figure image with click-to-zoom | `${IMG('1', 'Figure 1 — My Chart')}` |

### Adding a Bullet List

```js
${BULLETS([
    ['blue-400',   '<strong>Point 1:</strong> Your text here (Bret <em>et al.</em>, 2026).'],
    ['purple-400', '<strong>Point 2:</strong> Another point with <em>italics</em>.'],
    ['red-400',    '<strong>Key Finding:</strong> Important result.'],
])}
```
> **Note:** Always write `et al.` as `<em>et al.</em>` to get the correct italic formatting.

### Available Icons

```js
ICON_BOLT    // Lightning bolt — for introduction/model slides
ICON_DB      // Database cylinder — for dataset slides
ICON_CHART   // Bar chart — for results/figure slides
ICON_DNA     // DNA helix — for structural biology slides
ICON_WARN    // Warning triangle — for paradox/anomaly slides
ICON_STAR    // Star — for conclusion/insight slides
ICON_BOOK    // Open book — for references slides
```

## 🖼️ How to Add a New Figure Slide

**Step 1 — Add your image**
Name your file `Figure N.jpg` (where N is the figure number) and place it in:
```
ppt/assets/Figure N.jpg
```
> Accepted formats: `.jpg`, `.jpeg`, `.png`.
> Images should have a **white background** to match the light theme.

**Step 2 — Add the slide in `content.js`**

Copy this template and add it at the correct position in the `slidesData` array:

```js
// ─── SLIDE N ─────────────────────────────────────────────────────
{
    id: N, title: "Your Slide Title",
    notes: "What you will say during this slide.",
    content: [], figure: null,
    htmlContent: `
    ${PANEL(`
        ${HDR(ICON_CHART, 'Section X · Slide N', 'Your Slide Title', 'indigo-400', 'blue-400')}
        <div class="flex gap-12 h-full items-center">
            <div class="w-[40%] flex flex-col justify-center gap-6 z-10 shrink-0">
                ${BULLETS([
                ['blue-400',   '<strong>X-axis:</strong> What it represents.'],
                ['indigo-400', '<strong>Y-axis:</strong> What it measures.'],
                ['cyan-400',   'Key finding from this figure (Author <em>et al.</em>, 2026).'],
            ])}
            </div>
            <div class="w-[60%] h-full flex flex-col items-center justify-center relative min-h-0 min-w-0 pb-6">
                ${IMG('N', 'Figure N — Your Figure Caption')}
            </div>
        </div>
    `)}`
},
```

**Step 3 — Update slide IDs**
Make sure every slide's `id:` field matches its position (1 → 2 → 3...).

## 🖼️ How to Change the Background Image

The 3D molecular image shown behind the slides and dashboard is located at:
```
ppt/assets/bg.png
```

To replace it:
1. Find or create a new image (preferably a high-resolution, light-colored image)
2. Name it `bg.png`
3. Replace the file in `ppt/assets/`
4. Refresh the browser — changes apply immediately

The image is referenced in `index.html` line ~66:
```html
<body style="background-image: url('./ppt/assets/bg.png'); ...">
```

    backdrop-filter: blur(35px);               /* Frosted glass blur */
}

## 📊 How the Live Preview Works (No Thumbnails Needed)

The dashboard previews (sidebar thumbnails + main preview window) use **live HTML rendering**.
The actual slide HTML is rendered in a downscaled container — no `.webp` thumbnail files are required.

This is handled in `presentationEngine.js → renderDashboardPreview()`:
- Sidebar thumbnails: scaled to ~17.5% of full size
- Main preview panel: scaled to ~31.5% of full size
- Presenter HUD next-slide: scaled to ~22% of full size

## ⚠️ Important Rules

1. **Do NOT edit `tailwind-build.css`** — it is auto-generated and will be overwritten
2. **Image filenames are case-sensitive** — use exactly `Figure 1.jpg`, `Figure 2.jpg` etc.
3. **Keep slide IDs sequential** — gaps in IDs cause rendering bugs in the engine
4. **Always use backtick strings** for `htmlContent` — never single or double quotes
5. **`et al.` must be italic** — always write `<em>et al.</em>` in slide text
6. **Hyperlinks open in new tab** — always add `target="_blank"` to `<a>` tags

## 🏗️ Architecture Summary

```
User Action
    │
    ▼
index.html (3 views: dashboard, stage, presenter)
    │
    ▼
app.js → boots PresentationEngine + feature modules
    │
    ├── presentationEngine.js
    │     ├── reads slidesData from content.js
    │     ├── renders slides via getSlideHTML()
    │     ├── scales slides via ResizeObserver + scaleCanvases()
    │     └── syncs across windows via BroadcastChannel
    │
    ├── content.js
    │     └── exports slidesData[] array
    │           └── each slide has htmlContent (raw HTML string)
    │
    ├── keyboard.js   — arrow key navigation
    ├── fullscreen.js — Fullscreen API
    └── cursor.js     — custom cursor effects
```
